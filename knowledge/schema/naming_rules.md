# 命名规范

## 目标
统一多 sensor / 多 ISP / 多厂家场景下的页面命名，保证命名简洁、稳定、便于检索。

## raw 文件
格式建议：
- `主体_资料类型_标题.md/pdf`
- 主体可以是 sensor、ISP、厂家或方案名

例如：
- `ISX031_应用手册.pdf`
- `Sony_HDR资料摘录.md`
- `OX03C10_XXXISP_联调笔记.md`

## wiki 文件
### 平台页
平台页统一放在 `wiki/platforms/`，不依赖更深子目录。

例如：
- `ISX031.md`
- `Sony.md`
- `XXX_ISP.md`
- `ISX031_XXX_ISP方案.md`

说明：
- 更细的角色区分通过页面属性表达
- 页面属性可标记主类型为 Sensor / ISP / Solution / Vendor

### 模块页
分两类命名：
- 通用模块页：`AE.md`、`AWB.md`、`HDR.md`
- 平台专属模块页：`ISX031_AE.md`、`ISX031_AWB.md`、`ISX031_HDR.md`

说明：
- 若页面内容主要来自单一平台资料，且寄存器、实现方式、调试顺序明显依赖该平台，优先使用“平台前缀 + 模块名”命名
- 若后续抽象出了跨平台仍成立的共性知识，再单独整理为不带平台前缀的通用模块页
- 不要把单个平台内容继续伪装成纯通用模块名，否则后续多平台容易混淆

### 问题页
- `偏色.md`
- `噪声大.md`
- `高光过曝.md`

### 工具页
- `某某调试工具.md`

### 流程页
- 通用流程：`偏色调试流程.md`
- 平台流程：`ISX031_图像质量调整流程.md`

### 调试案例 / 项目记录
调试案例优先写入对应问题页的小节，不再默认新建独立案例页。

例如：
- `偏色.md` 中增加 `## 调试案例 / 项目记录`
- `HDR接缝和运动伪影.md` 中记录某平台 HDR 接缝案例

只有当一个案例横跨多个问题、篇幅很长且需要独立引用时，才考虑新建独立页面，并在相关问题页建立链接。

### 平台索引页
- `ISX031_索引.md`

## logs 文件
日志文件建议按月份组织：
- `logs/YYYY-MM.md`

例如：
- `logs/2026-04.md`

说明：
- 摄取日志和 Lint 审查记录可以写在同一个月度日志中
- 小修小补不强制写日志
- 批量摄取、索引重构、页面合并、定期审查建议写日志
