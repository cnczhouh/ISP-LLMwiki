# ISP 知识入库工作流

## 1. 资料摄入
将原始资料放入 `raw/`：
- 平台手册
- 应用手册
- datasheet
- 工具说明
- 调试记录
- 经验总结

## 2. 初步识别
从 raw 中识别：
- 厂家
- 主体对象（sensor / ISP / 方案 / 厂家）
- 模块
- 问题
- 工具
- 流程
- 可沉淀案例

## 3. 先判断页面角色
先判断资料更适合进入哪类页面：
- 通用模块 / 通用问题 / 通用流程 / 通用工具
- 平台页
- 对应问题页中的调试案例 / 项目记录
- 索引页

## 4. 页面属性标注
建页或补页时，优先补页面属性：
- 类型
- 主类型
- 附加属性
- 厂家
- 平台
- 模块
- 场景
- 适用范围

## 5. 知识整理
整理进入 `wiki/` 时遵循：
- 先抽主干答案，再补平台入口
- 先查是否已有页面可增量补充
- 若是平台实现或平台使用方法，优先写平台页
- 若是某次项目经验，优先沉淀到对应问题页的“调试案例 / 项目记录”小节

## 6. 建立关联
每个页面至少关联：
- 一个相关平台或主体对象
- 一个相关模块、问题或流程
- 一个来源页

## 7. 增量维护
后续新增资料时：
- 先查是否已有对应页面
- 优先更新，不重复创建
- 若资料能沉淀成稳定方法，补入通用页
- 若资料主要回答某平台怎么用，补入平台页
