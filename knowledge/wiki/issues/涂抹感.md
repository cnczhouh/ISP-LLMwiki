# 涂抹感

涂抹感是指画面在降噪或多帧处理后，纹理和边缘像被抹开一样发糊、发腻，常见于低照度场景。

- 类型：通用问题
- 厂家：跨厂家 / Sony / SmartSens / GEO Semiconductor / Fullhan

- 模块：[[wiki/modules/ISX031_NR|NR]] / [[wiki/modules/ISX031_Sharpen|Sharpen]] / [[wiki/modules/SC360AT_NR|SC360AT_NR]] / [[wiki/modules/SC360AT_Sharpness|SC360AT_Sharpness]] / [[wiki/modules/GW5_SNR|GW5_SNR]] / [[wiki/modules/GW5_CNR|GW5_CNR]] / [[wiki/modules/GW5_LTM|GW5_LTM]] / [[wiki/modules/FH833X_NR|FH833X_NR]] / [[wiki/modules/FH833X_YNR|FH833X_YNR]] / [[wiki/modules/FH833X_CNR|FH833X_CNR]] / [[wiki/modules/FH833X_SHARP_NOISE|FH833X_SHARP_NOISE]]
- 场景：低照度、高增益、降噪过重、动态场景

## 现象表现
- 画面干净但像被磨皮
- 纹理边界发糊、发腻
- 动态场景更容易出现糊成一片的观感

## 常见触发场景
- NR 过强
- 时域处理或多帧处理过重
- 为压低照噪声而牺牲了纹理保真

## 优先排查顺序
1. 先检查 [[wiki/modules/ISX031_NR|NR]] 的 RAW 与色度降噪强度。
2. 再看 [[wiki/modules/ISX031_Sharpen|Sharpen]] 是否试图把被抹掉的纹理硬拉回来，导致更假。
3. 在动态场景下复查，确认是否伴随拖尾或局部重影。

## 关键观察项
- 皮肤和墙面是否塑料感明显
- 纹理边界是否发糊
- 动态场景是否比静态图更明显

## 常见误判
- 把“画面更干净”直接理解成“调得更好”
- 用更强锐化去补救涂抹感，结果引入假边

## 平台差异入口
- 不同平台的涂抹感可能来自 RAW 降噪、时域降噪、HDR 合成或后级锐化补偿策略。
- 相关平台页：[[wiki/platforms/ISX031|ISX031]]

- SC121AT：优先关联 [[wiki/modules/SC121AT_NR|SC121AT_NR]]、[[wiki/modules/SC121AT_Sharpness|SC121AT_Sharpness]] 与 HDR 合成。低照或动态场景下若画面发腻，先降低/分档检查 NR，再确认锐化没有用假细节掩盖涂抹。
- SC360AT：优先关联 [[wiki/modules/SC360AT_NR|SC360AT_NR]] 和 [[wiki/modules/SC360AT_Sharpness|SC360AT_Sharpness]]。低照涂抹先按 HCG / LCG / S / VS 分支看 DNS13 / DNS23 是否过强，再检查边缘判断、denoise strength、add back、YDNS 与 DnsBack；若画面干净但纹理塑料感重，避免继续加锐化硬补，应先放松 NR 或提高细节 add back。
- GW5：优先关联 [[wiki/modules/GW5_SNR|GW5_SNR]]、[[wiki/modules/GW5_CNR|GW5_CNR]]、[[wiki/modules/GW5_Demosaic|GW5_Demosaic]] 和 [[wiki/modules/GW5_LTM|GW5_LTM]]。低照高 gain 下若 LTM 抬暗放大噪声，NR 可能被迫加重并形成涂抹，需要联动检查 AE gain、LTM strength / slope 与 SNR/CNR 分档。
- FH833X：优先关联 [[wiki/modules/FH833X_NR|FH833X_NR]]、[[wiki/modules/FH833X_YNR|FH833X_YNR]]、[[wiki/modules/FH833X_CNR|FH833X_CNR]]、[[wiki/modules/FH833X_APC|FH833X_APC]] 和 [[wiki/modules/FH833X_SHARP_NOISE|FH833X_SHARP_NOISE]]。强 NR3D 方案要重点观察运动区域拖影和纹理释放；弱 3DNR 车载场景应避免 NR2D / YNR 每级过强，把运动画面抹成一片。
