# 输出格式错误

输出格式错误是指 sensor / ISP 输出的 RAW、YUV、bit depth、Bayer order、MIPI data type、virtual channel 或字节排列与接收端解析不一致。

## 页面属性
- 类型：通用问题
- 厂家：跨厂家
- 平台：跨平台
- 模块：Output Format / RAW / YUV / MIPI / DVP / LVDS
- 场景：点亮 bring-up、格式切换、HDR VC、RAW / YUV 输出调试
- 适用范围：跨平台

## 现象表现
- RAW 颜色严重异常，红蓝反、绿通道错或 Bayer pattern 不对。
- YUV 画面偏色、紫绿、亮度正常但色度错误。
- bit depth 不匹配导致画面过暗、过亮或灰阶断裂。
- HDR 多 VC 输出时接收端取错通道或帧序。

## 优先排查顺序
1. 确认 sensor 端配置的 RAW / YUV 输出格式和 bit depth。
2. 检查接收端 Bayer order、unpack、stride、endian 是否匹配。
3. MIPI 场景检查 data type、virtual channel、lane 数和 HDR 分路输出。
4. YUV 场景确认 YUYV / UYVY / VYUY / YVYU 排列。
5. 若尺寸或 crop 同时异常，联查 [[wiki/issues/分辨率和裁剪异常|分辨率和裁剪异常]]。

## 常见处理方向
- 从 RAW10 / RAW12 等明确格式开始验证，再打开 HDR VC 或 YUV 输出。
- 保存原始帧 dump，用离线工具按不同 Bayer order / bit depth 试解。
- 接收端和调试工具都要确认格式，避免 sensor 输出正确但显示工具解析错误。
