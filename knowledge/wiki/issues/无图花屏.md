# 无图花屏

无图花屏是指 sensor 或 ISP 链路没有正常输出可解析图像，表现为黑屏、无帧、花屏、乱码、颜色通道错乱或画面撕裂。

## 页面属性
- 类型：通用问题
- 厂家：跨厂家
- 平台：跨平台
- 模块：Sensor Bring-up / MIPI / DVP / LVDS / Output Format / Clock
- 场景：点亮 bring-up、接口调试、接收端配置、格式切换
- 适用范围：跨平台

## 现象表现
- 接收端无帧或帧率为 0。
- 有帧但画面花屏、错位、颜色异常或滚动。
- RAW 数据无法按预期 Bayer pattern 解析。
- YUV 输出颜色通道异常或同步错乱。

## 优先排查顺序
1. 先确认供电、时钟、reset、I2C 通信和 sensor ID。
2. 检查 sensor 是否进入 streaming，输出帧率是否符合预期。
3. 检查接口类型、lane 数、bit depth、MIPI data type / virtual channel 或 DVP / LVDS 时序。
4. 检查输出格式、分辨率、line length、frame length 和接收端 stride / unpack 配置。
5. 若只有局部错位或尺寸异常，联查 [[wiki/issues/分辨率和裁剪异常|分辨率和裁剪异常]]。

## 常见处理方向
- 先用最简单输出模式点亮，再逐步打开 HDR、压缩、裁剪和 ISP 功能。
- sensor 端输出格式和接收端解析格式必须逐项对齐。
- MIPI / LVDS 问题要同时看硬件连接、lane mapping、速率和接收端时钟配置。
- RAW 花屏时优先保存原始 dump，避免显示工具二次处理掩盖问题。
