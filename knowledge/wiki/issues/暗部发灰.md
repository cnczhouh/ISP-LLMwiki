# 暗部发灰

暗部发灰是指画面黑位不扎实、阴影区像蒙了一层灰，常见于黑电平基线不稳、Gamma/ATR 抬升过度或主观亮度风格调得过头的场景。

- 适用范围：当前已覆盖 [[wiki/platforms/ISX031|ISX031]]、[[wiki/platforms/SC121AT|SC121AT]] 与 [[wiki/platforms/SC361AT|SC361AT]]，后续可继续扩展为跨平台暗部发灰排查知识

## 现象表现
- 黑位不扎实，阴影区泛灰
- 暗部看似有细节，但整体脏、松、浮
- 高对比场景里暗部缺少应有的沉稳感

## 常见触发场景
- 黑电平基线不稳
- 暗场补偿或亮度映射过度抬升
- 对比度和亮度主观风格调节过猛

## 优先排查顺序
1. 先确认黑电平基线是否稳定，检查 [[wiki/modules/ISX031_Clamp(BLC)|Clamp(BLC)]]。
2. 若暗部问题带固定区域特征，回看 [[wiki/modules/ISX031_Dark Shading Compensation|Dark Shading Compensation]]。
3. 检查 [[wiki/modules/ISX031_Gamma|Gamma]] 和 [[wiki/modules/ISX031_ATR|ATR]] 是否把暗部层次抬得过高。
4. 若只是主观观感偏灰，检查 [[wiki/modules/ISX031_Brightness Adjustment|Brightness Adjustment]] 的对比度与 Y offset 调整。

## 关键观察项
- 黑位是否稳定
- 暗部是否带固定空间分布异常
- 中间调是否被连带抬灰
- 白天与夜晚是否都发灰

## 常见误判
- 把黑位基线问题误判成单纯 Gamma 风格问题
- 只看暗部细节变多，就误以为调优变好

## 平台差异入口
- 不同平台里，暗部发灰可能来自前级黑位补偿、tone mapping、局部对比度增强或主观风格层。
- 若后续接入新平台，优先补该平台的黑电平处理链路和亮度映射链路，再扩展到问题页。
- 相关平台页：[[wiki/platforms/ISX031|ISX031]]

- SC121AT：优先关联 Tonemapping、[[wiki/modules/SC121AT_Gamma|SC121AT_Gamma]]、[[wiki/modules/SC121AT_Contrast|SC121AT_Contrast]] 与 [[wiki/modules/SC121AT_LSC|SC121AT_LSC]]。若暗部整体发灰，先区分是黑位/LSC 基础问题，还是 tone mapping 与 Gamma 抬暗部过度；低照下还要注意 LSC gain 衰减和噪声抬升。
- SC360AT：优先关联 [[wiki/modules/SC360AT_HDR|SC360AT_HDR]]、[[wiki/modules/SC360AT_Gamma|SC360AT_Gamma]]、[[wiki/modules/SC360AT_Contrast|SC360AT_Contrast]] 和 [[wiki/modules/SC360AT_LSC|SC360AT_LSC]]。若暗部发灰但曝光落点正常，重点检查 Tonemapping 是否抬暗部过度、Gamma 暗部曲线是否过高、Contrast Low Level / Stretch 是否让黑位变浮；若暗角或边缘暗部发灰，再看 LSC 校准与高 gain Ratio Down。
- SC361AT：优先关联 [[wiki/modules/SC361AT_HDR|SC361AT_HDR]]、[[wiki/modules/SC361AT_AEC_AGC|SC361AT_AEC_AGC]] 和 [[wiki/modules/SC361AT_PWL|SC361AT_PWL]]。若暗部发灰或中暗调浮起，先区分是 HDR 合成后 PWL / LTM 映射抬暗过度，还是低亮分支曝光 / gain 不足导致的灰脏感。
- GW5：优先关联 Black Offset、Gamma 和 [[wiki/modules/GW5_LTM|GW5_LTM]]。若 LTM 的 `strength_inroi`、暗部 asymmetry curve 或 `slope_max` 过强，暗部细节会变多但黑位可能变浮，并伴随噪声可见性上升。
- FH833X：优先关联 [[wiki/modules/FH833X_BLC|FH833X_BLC]]、[[wiki/modules/FH833X_Gamma|FH833X_Gamma]]、[[wiki/modules/FH833X_DRC|FH833X_DRC]]、[[wiki/modules/FH833X_LCE|FH833X_LCE]] 和 [[wiki/modules/FH833X_YIE|FH833X_YIE]]。先确认 BLC 基线和 gain 联动，再看 DRC curve 暗部是否拉太高、`bright` 是否频繁变化、`lcCtr` 和暗区细节增益是否导致黑位变浮，最后检查暗区饱和度抑制。
