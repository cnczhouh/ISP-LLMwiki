# 高光过曝

高光过曝表现为亮部细节丢失、天空或灯具区域发白，严重时会破坏整体层次。

## 页面属性
- 类型：通用问题
- 厂家：跨厂家 / Sony / SmartSens / GEO / Fullhan
- 平台：[[wiki/platforms/ISX031|ISX031]] / [[wiki/platforms/SC121AT|SC121AT]] / [[wiki/platforms/SC360AT|SC360AT]] / [[wiki/platforms/SC361AT|SC361AT]] / [[wiki/platforms/GW5|GW5]] / [[wiki/platforms/FH833X|FH833X]]
- 模块：[[wiki/modules/ISX031_AE|AE]] / [[wiki/modules/ISX031_HDR|HDR]] / [[wiki/modules/ISX031_ATR|ATR]] / [[wiki/modules/ISX031_Gamma|Gamma]] / [[wiki/modules/SC360AT_AE|SC360AT_AE]] / [[wiki/modules/SC360AT_HDR|SC360AT_HDR]] / [[wiki/modules/SC360AT_Gamma|SC360AT_Gamma]] / [[wiki/modules/SC361AT_HDR|SC361AT_HDR]] / [[wiki/modules/SC361AT_AEC_AGC|SC361AT_AEC_AGC]] / [[wiki/modules/SC361AT_PWL|SC361AT_PWL]] / [[wiki/modules/FH833X_AE|FH833X_AE]] / [[wiki/modules/FH833X_WDR|FH833X_WDR]] / [[wiki/modules/FH833X_DRC|FH833X_DRC]]
- 场景：强光场景、逆光场景、高反差场景、高亮区域细节丢失
- 适用范围：当前已覆盖 [[wiki/platforms/ISX031|ISX031]]、[[wiki/platforms/SC121AT|SC121AT]] 与 [[wiki/platforms/SC361AT|SC361AT]]，后续可继续扩展为跨平台高光过曝排查知识

## 现象描述
- 亮部大片发白
- 灯具或反光区域没有层次
- 高亮区域边缘可能伴随偏色或复合异常

## 排查顺序
1. 先检查 [[wiki/modules/ISX031_AE|AE]] 的目标曝光、测光模式和高光权重。
2. 若是大光比场景，检查 [[wiki/modules/ISX031_HDR|HDR]] 的阈值与合成设置。
3. 确认 [[wiki/modules/ISX031_ATR|ATR]] 是否对高光压缩不足。
4. 检查 [[wiki/modules/ISX031_Gamma|Gamma]] 是否让高光层次过早顶死。

## 典型判断依据
- RAW 是否已饱和
- 高光问题是整幅过亮还是局部层次不够
- 复合区是否伴随颜色偏移

## 平台差异入口
- 不同平台对高光的处理可能分别落在曝光策略、HDR 合成、tone mapping 或传感器饱和特性上，不能只从 [[wiki/modules/ISX031_AE|AE]] 单点判断。
- 若后续接入新平台，优先补该平台的动态范围能力、HDR 实现方式和高光压缩链路，再回写到问题页。

- SC121AT：优先关联 [[wiki/modules/SC121AT_AE|SC121AT_AE]]、[[wiki/modules/SC121AT_HDR|SC121AT_HDR]] 与 ISPC_HDR。先判断 RAW 或 HDR 合成前数据是否已经饱和，再检查行交叠 HDR 的曝光比例、合成阈值和 tone mapping 是否保留高光层次。
- SC360AT：优先关联 [[wiki/modules/SC360AT_AE|SC360AT_AE]] 和 [[wiki/modules/SC360AT_HDR|SC360AT_HDR]]。先分 HCG / LCG / S / VS 看高光是否已饱和，再检查 VS 短曝光是否保住高光、HDR Combine 的 BloomingMode / HighMargin / BloomingShift 和 Tonemapping 的 Dynamic Range / Global Gamma / Local Tonemapping 是否压住亮部层次；若启用 ISPC_HDR，还要确认高光相关参数是否被接管。
- SC361AT：优先关联 [[wiki/modules/SC361AT_HDR|SC361AT_HDR]]、[[wiki/modules/SC361AT_AEC_AGC|SC361AT_AEC_AGC]] 和 [[wiki/modules/SC361AT_PWL|SC361AT_PWL]]。该平台当前重点是 PixGain HDR / Lofic HDR、`LE-OF`、`SE` 与 PWL / LTM，高光过曝时先判断 OF / SE 路是否有效参与合成，再看 SE 曝光、OF gain 和 PWL 高亮段压缩是否保留高亮层次。
- GW5：优先关联 [[wiki/modules/GW5_AE|GW5_AE]] 与 [[wiki/modules/GW5_LTM|GW5_LTM]]。先判断输入或 AE target 是否已经让高光饱和；若未饱和，再检查 LTM 的 `white_level`、高光 asymmetry LUT、`slope_min` 和 strength 是否保留高光层次。
- FH833X：优先关联 [[wiki/modules/FH833X_AE|FH833X_AE]]、[[wiki/modules/FH833X_WDR|FH833X_WDR]] 和 [[wiki/modules/FH833X_DRC|FH833X_DRC]]。WDR 下先单出长 / 中 / 短帧确认短帧是否保住高光，再看曝光比、merge 曲线、HLR、`sensor_output_max` 和 DRC 高亮压缩。
