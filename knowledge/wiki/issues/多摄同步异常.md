# 多摄同步异常

多摄同步异常是指多颗 sensor 或多路摄像头之间曝光时刻、帧率、触发、亮度状态或输出帧序不一致，导致拼接、环视、双目或多摄融合出现不稳定。

## 页面属性
- 类型：通用问题
- 厂家：跨厂家 / SmartSens / Sony
- 平台：跨平台
- 模块：Sync / Slave Mode / FSYNC / AE / Frame Rate
- 场景：车载环视、双目、多摄拼接、外触发同步
- 适用范围：跨平台

## 现象表现
- 多路画面亮度不同步，一路先亮或先暗。
- 拼接区域运动物体错位。
- 多摄帧号、时间戳或输出节奏不一致。
- 外触发后部分 sensor 曝光时间或帧率异常。

## 优先排查顺序
1. 确认主从模式、触发信号、帧率和输出时序是否一致。
2. 检查各路 sensor 曝光时间、gain、AE 状态是否同步或受同一策略控制。
3. 检查 FSYNC / EFSYNC / GPIO 触发宽度、周期和 active state 是否满足要求。
4. HDR 模式下确认各路长短曝光比例和 VC / frame order 是否一致。
5. 检查接收端是否按正确帧序、时间戳和虚拟通道解析。

## 常见处理方向
- 固定触发源和帧率，先让所有 sensor 输出节奏一致。
- 多摄拼接场景尽量统一 AE 策略或共享曝光信息。
- 缩短不合理的触发 active state，避免同一帧内曝光状态不一致。
- 接收端同步要同时看硬件触发、帧号、时间戳和曝光元数据。

## 平台差异入口
- SC121AT：优先关联 [[wiki/platforms/SC121AT|SC121AT]] 的 Slave Mode、EFSYNC / FSYNC / GPIO 触发和多传感器同步风险点。
- SC361AT：后续可结合 Embedded Data 和输出时序补充多摄同步检查入口。
- ISX031：优先检查平台同步模式、帧同步和 AE 控制权。
