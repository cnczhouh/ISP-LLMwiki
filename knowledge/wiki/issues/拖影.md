# 拖影

拖影通常出现在低照或曝光时间较长的场景，表现为运动物体边缘拉尾或动态模糊。

## 页面属性
- 类型：通用问题
- 厂家：跨厂家 / Sony / SmartSens / Fullhan
- 平台：[[wiki/platforms/ISX031|ISX031]] / [[wiki/platforms/SC121AT|SC121AT]] / [[wiki/platforms/FH833X|FH833X]]
- 模块：[[wiki/modules/ISX031_AE|AE]] / [[wiki/modules/ISX031_NR|NR]] / [[wiki/modules/ISX031_HDR|HDR]] / [[wiki/modules/ISX031_Sharpen|Sharpen]] / [[wiki/modules/FH833X_AE|FH833X_AE]] / [[wiki/modules/FH833X_NR|FH833X_NR]] / [[wiki/modules/FH833X_WDR|FH833X_WDR]]
- 场景：低照动态场景、运动主体、曝光时间过长、时域处理过强
- 适用范围：当前已覆盖 [[wiki/platforms/ISX031|ISX031]] 与 [[wiki/platforms/SC121AT|SC121AT]]，后续可继续扩展为跨平台拖影排查知识

## 现象描述
- 运动主体边缘拖尾
- 夜景走动人物糊
- 画面稳定时还好，动态时明显变差

## 排查顺序
1. 查看曝光时间是否过长。
2. 检查 AE 图是否允许在暗场下继续拉长快门。
3. 检查增益与快门的取舍是否合理。
4. 观察降噪是否进一步放大拖影观感。

## 典型判断依据
- 拖影主要来自曝光时间过长，还是来自多帧处理/降噪副作用
- 静止画面是否正常，只有动态主体才明显变差
- 高增益下是否伴随涂抹感、拖尾或局部重影

## 平台差异入口
- 不同平台的拖影来源可能落在曝光时间策略、HDR 多帧合成、时域降噪或锐化副作用，不能默认只看快门时间。
- 若后续接入新平台，优先补该平台的长曝光策略、多帧处理方式和动态场景降噪逻辑，再扩展到问题页。

- SC121AT：优先关联 [[wiki/modules/SC121AT_HDR|SC121AT_HDR]] 的 Staggered HDR、曝光比例和 HDR 合成策略。若动态主体拖尾，先区分是长曝光导致的运动模糊，还是多曝光 HDR 合成在运动边缘产生的重影。
- FH833X：优先关联 [[wiki/modules/FH833X_AE|FH833X_AE]]、[[wiki/modules/FH833X_NR|FH833X_NR]]、[[wiki/modules/FH833X_SHARP_NOISE|FH833X_SHARP_NOISE]] 和 [[wiki/modules/FH833X_WDR|FH833X_WDR]]。先缩短曝光时间确认是否改善，车载场景可用 AE 节点模式控制曝光；若来自 NR3D，降低时域降噪强度、调整 `mdGain`、slope / offset 和 `refCoeffRatio`；若仅高光合成区域拖影，则检查 WDR 多帧合成和 merge 区域。
