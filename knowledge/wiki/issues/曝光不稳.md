# 曝光不稳

曝光不稳是指画面亮度随时间来回跳变、AE 收敛慢、收敛后震荡，或场景轻微变化就引起明显明暗波动。

## 页面属性
- 类型：通用问题
- 厂家：跨厂家 / Sony / SmartSens / GEO / Fullhan
- 平台：跨平台
- 模块：AE / AEC / AGC / HDR / Antiflicker
- 场景：亮度跳变、AE 震荡、室内外切换、车载动态光照
- 适用范围：跨平台

## 现象表现
- 静止场景下画面亮度仍周期性或随机跳动。
- 从暗到亮、从亮到暗切换时收敛过慢或过冲。
- 局部强光进入画面后整体曝光被明显拉低。
- HDR 场景下长短曝光比例变化造成亮度突变。

## 优先排查顺序
1. 确认是否是光源频闪导致的亮度波动，先联查 [[wiki/issues/闪烁|闪烁]]。
2. 检查 AE target、dead band、hysteresis 是否过窄，避免小亮度变化反复触发调节。
3. 检查测光区域和权重，确认高亮区域、天空、车灯是否过度影响整体曝光。
4. 检查最大曝光、最小曝光、gain 上限和帧率切换阈值是否造成档位突变。
5. HDR 平台需同步检查长 / 中 / 短曝光比例和合成前后亮度是否连续。

## 常见处理方向
- 适当增大 AE dead band 或 hysteresis，降低轻微亮度变化导致的震荡。
- 优化 ROI / zone weight，避免无关高亮区域主导曝光。
- 调整 AE 收敛速度，使切换场景时不过冲、不拉风箱。
- 室内灯光场景需确认 antiflicker 频率和曝光行数约束。

## 平台差异入口
- SC121AT：优先关联 [[wiki/modules/SC121AT_AE|SC121AT_AE]]，检查 AE target、Antiflicker、ROI 和 HDR 曝光比例。
- SC360AT：优先关联 [[wiki/modules/SC360AT_AE|SC360AT_AE]] 和 [[wiki/modules/SC360AT_HDR|SC360AT_HDR]]。先看 HCG / LCG / S / VS 四帧 Mean 和 `USE_Exp.Mean`，再检查 AE target、Fast / Slow speed、进出稳定区 tolerance、ROI 权重，以及 ISPC_HDR 是否接管 AE_Target。
- SC361AT：优先关联 [[wiki/modules/SC361AT_AEC_AGC|SC361AT_AEC_AGC]]，检查 LE / SE 曝光、HCG / LCG / OF / SE gain 约束。
- GW5：优先关联 [[wiki/modules/GW5_AE|GW5_AE]]，检查 PID、dead band、frame rate switching、zone weight 和 HDR ratio。
- FH833X：优先关联 [[wiki/modules/FH833X_AE|FH833X_AE]]，检查 `lumaRef`、`tolerance`、`lightChangeZone/greatChangeZone`、AE 速率、16x16 统计权重，以及 intt / again / dgain 延迟是否同帧生效。
