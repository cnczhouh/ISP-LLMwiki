# 噪声大

噪声大通常表现为暗部颗粒明显、色块抖动或画面发脏，常见于高增益和低照场景。

## 现象描述
- 暗部颗粒感强
- 色噪明显
- 纯色区不干净
- 动态场景下噪声伴随拖影或涂抹

## 排查顺序
1. 确认曝光时间和增益是否过高。
2. 区分是亮度噪声还是色度噪声。
3. 检查 [[wiki/modules/ISX031_NR|NR]] 是否开启，以及强度是否合理。
4. 检查 [[wiki/modules/ISX031_Sharpen|Sharpen]] 是否把噪声一起增强。
5. 若出现在 HDR 复合区，检查 [[wiki/modules/ISX031_HDR|HDR]] 阈值和切换方法。

## 典型判断依据
- 暗部是否主要是亮度颗粒还是彩色噪点
- 纹理是否随着降噪一起消失
- 复合区是否有额外点噪或偏色

## 平台差异入口
- 不同平台的噪声主因可能落在 sensor 侧、前级补偿侧或 ISP 降噪侧，不能默认只怪 [[wiki/modules/ISX031_NR|NR]]。
- 若后续接入新平台，优先补该平台的增益策略、HDR 实现方式和降噪链路，再扩展到问题页。

- SC121AT：优先关联 [[wiki/modules/SC121AT_NR|SC121AT_NR]]、[[wiki/modules/SC121AT_AE|SC121AT_AE]] 的 gain 策略、[[wiki/modules/SC121AT_HTEMP|SC121AT_HTEMP]] 与 [[wiki/modules/SC121AT_LSC|SC121AT_LSC]]。高温或低照噪声上升时，先看增益和温度补偿，再看 NR 强度，避免只靠降噪掩盖 sensor 侧噪声。
- SC360AT：优先关联 [[wiki/modules/SC360AT_NR|SC360AT_NR]]、[[wiki/modules/SC360AT_Sharpness|SC360AT_Sharpness]]、[[wiki/modules/SC360AT_AE|SC360AT_AE]] 和 [[wiki/modules/SC360AT_HDR|SC360AT_HDR]]。先按 HCG / LCG / S / VS 分支区分噪声来源，再看 gain node、DNS13 / DNS23、边缘判断、G 与 BR 通道强度比例和 add back；若噪声被锐化放大，联查 Sharpness / YUVDNS 的 YDNS、DnsBack、UVDNS 与边缘阈值。
- SC361AT：优先关联 [[wiki/modules/SC361AT_HDR|SC361AT_HDR]]、[[wiki/modules/SC361AT_AEC_AGC|SC361AT_AEC_AGC]] 和 [[wiki/modules/SC361AT_DPC|SC361AT_DPC]]。暗部噪声优先看 `LE-HCG` 低亮分支、曝光 / gain 是否已到上限，以及 HDR 合成后的映射策略；若是固定亮暗点或高温热噪点，再检查 DPC 是否开启并区分随机噪声和固定异常点。
- GW5：优先关联 Spatial Noise Reduction、Color Noise Reduction、Demosaic / Sharpening 与 [[wiki/modules/GW5_LTM|GW5_LTM]]。低照下 LTM 暗部增益与 `slope_max` 过强会让噪声更可见，需结合 strength、variance、slope 限制和 NR 一起判断。
- FH833X：优先关联 [[wiki/modules/FH833X_NR|FH833X_NR]]、[[wiki/modules/FH833X_YNR|FH833X_YNR]]、[[wiki/modules/FH833X_CNR|FH833X_CNR]]、[[wiki/modules/FH833X_DPC|FH833X_DPC]]、[[wiki/modules/FH833X_DRC|FH833X_DRC]] 和 [[wiki/modules/FH833X_APC|FH833X_APC]]。低照先用 NR3D `showCoefEn` 区分运动 / 静止，标定 slope / offset / `mdGain`；WDR 亮区噪声多来自中 / 短帧，需加强中短帧 NR2D / NR3D 并检查 WDR offset 和 merge 曲线；若噪声颗粒被放大，再看 DRC 细节增益和 APC 锐化。
