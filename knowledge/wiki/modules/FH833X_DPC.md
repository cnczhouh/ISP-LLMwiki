# FH833X_DPC

FH833X DPC（Defect Pixel Correction）坏点校正模块，用于处理 sensor 工艺、低照高增益等条件下出现的静态 / 动态白点和黑点。

## 页面属性
- 类型：平台模块
- 厂家：Fullhan / 富瀚微
- 平台：[[wiki/platforms/FH833X|FH833X]]
- 模块：DPC / 坏点校正
- 场景：固定亮点暗点、低照坏点、WDR 中短帧坏点、清晰度损失权衡
- 适用范围：指定平台

## 作用
- 校正 sensor 的坏点，包括孤立位置固定的静态坏点，以及低照高 gain 下随机出现的动态坏点。
- 在低照场景对图像整体观感影响较大，但强度过高会损伤分辨率和细节。

## 关键参数
- `dpc_en`：DPC 策略总开关。
- `mode`：manual / gainMapping。
- `ctrl_mode`：单个坏点校正 / 多个点校正。
- `enable`：白点、黑点或同时校正。
- `medBlendEn`：中值滤波使能。
- `supTwinkleEn`：闪烁坏点抑制。
- `mfStrCtrl`：中 / 短帧 DPC 强度单独控制。
- `dpcWTh0/1`：白点检测阈值，值越小检测越强，但更容易损失分辨率。
- `dpcBTh0/1`：黑点检测阈值，值越小检测越强。
- `medBlendRaito`、`medBlendStr`：中值校正融合比例和强度。
- `supTwinkleThl/Thh`：闪烁抑制阈值，用于极低照清晰度和闪烁坏点权衡。
- `mfStrRatio`：中 / 短帧强度比例。

## 调试步骤
1. 正常照度下可先使用单个坏点校正；低照可考虑多个坏点校正。
2. 初始可配置 `medBlendRatio=0x80`，即中值与去坏点校正约 1:1。
3. 初始 `medBlendStr=0x80`，低照可适当加大。
4. 若坏点仍存在，先调小白点 / 黑点阈值 0；仍无法去除时再调小阈值 1。
5. 切换到 gainMapping，在不同照度 / gain 下分别调整。
6. 极低照下如果清晰度损失明显或坏点闪烁明显，可评估 `supTwinkleEn`，但通常建议关闭。
7. WDR 中 / 短帧坏点更多时，可开启 `mfStrCtrl` 单独加强。

## 调试权衡
DPC 调试要区分坏点、随机噪声和真实细节，否则容易把纹理当坏点抹掉，或把低照噪声误判成坏点残留。

- 白点 / 黑点阈值越低，检测越敏感，孤立亮点暗点越容易消失，但高频纹理、星点、细线和边缘也更容易被误修。
- 多点校正比单点校正更适合低照高 gain 的密集坏点，但更容易损伤分辨率和细碎纹理。
- `medBlendEn`、`medBlendRaito`、`medBlendStr` 决定校正结果与中值的融合程度。中值占比越高，坏点更不明显，但局部细节更容易被抹平。
- `supTwinkleEn` 可压极低照闪烁坏点，但如果阈值过强，会把随机噪声和细节一起稳定成发糊的块状观感。
- WDR 中 / 短帧通常更噪，`mfStrCtrl` 可单独加强，但会影响高光合成区域的细节和边缘自然度。
- DPC、NR 和 APC 是串联关系：DPC 漏掉的坏点会被 APC 放大，DPC 过强造成的细节损失也无法靠后级锐化完全恢复。

### 典型调试取舍
| 目标 | 常见调整方向 | 图像收益 | 主要副作用 |
| --- | --- | --- | --- |
| 坏点去得更干净 | 降低检测阈值，增强中值融合 | 固定亮点 / 暗点减少 | 纹理和分辨率损失 |
| 保留细节 | 提高阈值，弱化多点和中值融合 | 高频纹理更自然 | 低照坏点可能残留 |
| 压极低照闪点 | 评估 `supTwinkleEn` 和阈值 | 闪烁坏点减少 | 局部发糊、块感增加 |
| WDR 短帧更干净 | 单独加强中 / 短帧 DPC | 高光合成坏点减少 | merge 区域细节损失 |
