# SC360AT_CCM

SC360AT CCM 模块提供高 / 中 / 低色温三组颜色校正矩阵，并按色温阈值进行矩阵选择和线性插值，用于在 AWB 后修正 sensor 色彩响应和主观色彩表现。

## 页面属性
- 类型：平台模块
- 厂家：SmartSens / 思特威
- 平台：[[wiki/platforms/SC360AT|SC360AT]]
- 模块：CCM / Color Correction Matrix
- 场景：高 / 中 / 低色温矩阵、色温阈值、矩阵插值、行归一化、饱和度后色彩微调
- 适用范围：指定平台

## 模块作用
- 提供高色温、中色温、低色温三组 CCM 矩阵。
- 根据当前色温和 CT 阈值选择 CCML、CCMM、CCMH 或在相邻矩阵之间线性插值。
- 资料建议一般先调整饱和度，再微调 CCM 矩阵。

## 关键参数和功能
- `Import_Export`：导入 / 导出 CCM 矩阵参数文件。
- `Save Data`：保存 CCM 矩阵参数到本地。
- `Load Data`：从本地导入 CCM 矩阵并写入寄存器。
- `Auto Normalization`：自动调整高中低三色温 CCM 矩阵每一行相加等于 1。
- `CT`：设置色温阈值。
- `CCM_Read`：读出当前 CCM 插值结果。

## CT 插值逻辑
- 当前色温 > L 时，选择 CCML。
- 当前色温 < H 时，选择 CCMH。
- 当前色温在 H 和 M 之间时，CCMH 与 CCMM 线性插值。
- 当前色温在 L 和 M 之间时，CCML 与 CCMM 线性插值。

## 调试视角
SC360AT CCM 应作为 AWB 和 Saturation 之后的颜色校正 / 微调模块，不适合用来掩盖 AWB 白点错误或 LSC shading 问题。

- 三组矩阵决定不同色温下的颜色风格，阈值设置不合理会造成色温切换时颜色跳变。
- 行归一化是保持白平衡不被矩阵破坏的基础；每行和偏离 1 时，灰阶和白色会被重新染色。
- 矩阵增强饱和度会让颜色更鲜艳，但也会放大低照色噪，尤其在 Saturation 已经拉高时更明显。
- 调某个色块的 hue 往往会牵连相邻颜色，例如红、橙、黄和肤色之间需要取舍。
- 低照下如果颜色噪声明显，应联查 Saturation 降饱和和 Sharpness / UVDNS，而不是只降低 CCM。

### 典型调试取舍
| 目标 | 常见调整方向 | 图像收益 | 主要副作用 |
| --- | --- | --- | --- |
| 色彩更准 | 使用三色温矩阵并保持行归一化 | 色卡和中性色更稳 | 主观记忆色未必讨好 |
| 色温切换更平滑 | 调 CT 阈值和矩阵差异 | 减少跳色 | 色温区间过宽时局部颜色不够准 |
| 色彩更浓 | 增强矩阵分离度或配合 Saturation | 饱和度提升 | 色噪、溢色、肤色偏差增加 |
| 低照更稳 | 减少矩阵过强和后级饱和度 | 彩噪下降 | 夜景颜色变淡 |

## 调试步骤
1. 先完成 AWB 标定并确认白点识别正确。
2. 先调 Saturation 到整体饱和度合理，再微调 CCM。
3. 配置高 / 中 / 低色温三组矩阵和 CT 阈值。
4. 打开 Auto Normalization，保证矩阵每行和为 1。
5. 用 CCM_Read 读取当前插值结果，检查色温过渡是否平滑。
6. 分别在高色温、中色温、低色温和混合光场景回归肤色、白墙、红绿灯、蓝天、绿植。
