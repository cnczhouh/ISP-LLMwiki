# FH833X_YIE

FH833X YIE（Y Image Enhancement）亮度域图像增强模块，用于在 Y 域做亮度提升和对比度增强，是亮度 / 对比度补偿型辅助模块。

## 页面属性
- 类型：平台模块
- 厂家：Fullhan / 富瀚微
- 平台：[[wiki/platforms/FH833X|FH833X]]
- 模块：YIE / 亮度增强 / 对比度增强
- 场景：亮度补偿、对比度补偿、YC 后级主观微调
- 适用范围：指定平台

## 作用
- 在亮度域做亮度增强。
- 在亮度域做对比度增强。
- 手册定位为辅助模块，建议在 AE、Gamma、DRC、LCE 等亮度相关模块稳定后再调。

## 关键参数
- `yc_en`：YIE / YC 策略总开关。
- `brt_mode`：亮度增强 manual / gainMapping。
- `ie_brt`：亮度增强值，S8，有符号；默认 0。
- `ctr_mode`：对比度增强 manual / gainMapping。
- `ie_ctr`：对比度调整因子，U2.6，`0x40` 为 1 倍。

## 调试视角
YIE 是后级亮度 / 对比度微调模块，适合做小幅主观补偿，不适合替代 AE、Gamma、DRC 或 LCE。

- `ie_brt` 抬高会让全局亮度上升，暗部看起来更亮，但黑位被抬起后容易发灰、发蒙，低照噪声也更明显。
- `ie_brt` 降低能压黑位、增强通透感，但暗部细节会更早丢失，背光主体可能变黑。
- `ie_ctr` 提高会增强亮暗反差，画面更硬朗，但高光和暗部分别更容易被压到两端，细节损失和噪声颗粒都会更明显。
- `ie_ctr` 降低能柔化画面和压边缘生硬感，但会牺牲通透性，使画面发平。
- 如果需要分亮度段控制，应优先考虑 Gamma、DRC 或 LCE；YIE 更适合最终风格微调。

### 典型调试取舍
| 目标 | 常见调整方向 | 图像收益 | 主要副作用 |
| --- | --- | --- | --- |
| 整体更亮 | 小幅提高 `ie_brt` | 主体亮度提升 | 黑位发灰、噪声上升 |
| 画面更通透 | 小幅降低 `ie_brt` 或提高 `ie_ctr` | 对比更强 | 暗部 / 高光细节损失 |
| 画面更柔和 | 降低 `ie_ctr` | 过渡更软 | 发平、不够清晰 |

## 调试建议
1. 亮度增强默认 `ie_brt=0`，一般不调整。
2. 抬升 `ie_brt` 可能导致暗处抬亮、画面变蒙。
3. 减小 `ie_brt` 可能损失暗部细节。
4. 对比度默认 `ie_ctr=0x40`，只有对比度有特殊需求时再轻微增强或减弱。
5. `ie_ctr` 过大可能导致高亮过曝、暗处过黑，损失细节。
