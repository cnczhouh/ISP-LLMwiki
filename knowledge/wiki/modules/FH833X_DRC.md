# FH833X_DRC

FH833X DRC 模块用于动态范围压缩，在保留宽动态细节的同时，将高动态范围场景映射到可显示亮度范围。

## 页面属性
- 类型：平台模块
- 厂家：Fullhan / 富瀚微
- 平台：[[wiki/platforms/FH833X|FH833X]]
- 模块：DRC / 动态范围压缩
- 场景：亮度映射、局部对比度、通透性、暗部细节、高亮过渡、halo、饱和度校正
- 适用范围：指定平台

## 模块作用
- DRC 将图像分为基础层和细节层，对基础层做亮度映射，对细节层做增强。
- DRC 同时提供色度校正，用于平衡亮度处理对饱和度的影响。
- 在 FH833X 图像调优中，DRC 同时影响亮度、对比度、通透性、暗部噪声和亮暗过渡。

## 关键参数
### 基础控制
- `drc_en`：DRC 策略总开关。
- `drcMode`：manual 或 gainMapping。
- `GradRevBypass`：高反差梯度翻转抑制 bypass，通常配置为 0。

### 统计窗口
- `spaceWin0Size` / `spaceWin1Size`：统计 0 / 1 空域窗口；窗口越大局部对比度越弱，运动 halo 越不明显。
- `binWin0Size` / `binWin1Size`：统计 0 / 1 值域窗口；窗口越大对比度越强，但 halo 风险更高。
- `mergeX1/Y1/X2/Y2`：统计 0 和统计 1 的融合曲线，stat0 偏暗区，stat1 偏亮区。
- `win_refresh`：统计窗口配置刷新位。

### 滤波输出权重
- `filter0_X0/X1`、`filter0WgtY0/Y1`：统计 0 滤波输出权重。
- `filter_shift`、`filter1_X0/X1`、`filter1WgtY0/Y1`：统计 1 滤波输出权重。
- 滤波输出权重越大，越倾向使用滤波结果，画面对比度和通透性更强，但过渡可能更生硬。

### 亮度映射与曲线
- `bright`：图像亮度调整，默认 0x7f；建议稳定，不要频繁调整。
- `drcCurveMode`：内置曲线或自定义 `drc_kl_curve`。
- `autoCurveEn` / `autoCurveMode` / `drcCurveWgt`：两条曲线内插，用于不同照度下的 DRC 曲线平滑切换。
- DRC 曲线必须单调非递减，否则可能不生效或产生异常效果。
- `refresh`：曲线手动调整后需要刷新。

### 亮暗分割与细节
- `FwExpMode`、`X0~X2`、`Y0~Y2`：亮暗分割曲线，决定亮区和暗区权重。
- `dePosDarkGain` / `deNegDarkGain`：暗区正 / 负向细节增益。
- `dePosBrightGain` / `deNegBrightGain`：亮区正 / 负向细节增益。
- 细节增益越大，通透性和对比度越强，但也更容易放大噪声、白边或黑边。

### 高亮过渡与饱和度
- `supBrightTh`：亮区抑制门限。
- `supBrightSlope`：亮区抑制斜率。
- `darkSat`：暗区饱和度抑制，255 表示不抑制。
- `brightSat`：亮区饱和度抑制，255 表示不抑制。
- 调 CCM 前应确认 DRC 亮区 / 暗区饱和度参数未误抑制颜色。

### 局部对比
- `lcCtr`：局部对比度，值越大边界对比越强。
- `lcCtr` 会影响灯光周围黑边和噪声颗粒，WDR 模式建议比线性模式更保守。

## 调试视角
DRC 的实际观感来自亮度映射、统计窗口、滤波权重、亮暗分割、细节增益和饱和度补偿的组合。

- DRC 曲线抬暗部能看见更多暗部细节，但也会把黑位、暗部噪声和压缩感一起抬起来；高亮压缩越强，越容易保层次但也越可能发灰。
- 统计窗口越偏局部，局部对比越强，但 halo、局部闪烁和运动边缘不自然更明显；窗口越大，画面更稳但通透性弱。
- 滤波输出权重和 `lcCtr` 提高会增强通透性和边界反差，但灯光周围黑边、白边和噪声颗粒会增加。
- 暗 / 亮区细节增益能补层次，但会直接放大 NR 残留噪声，并和 APC 的锐化边缘叠加。
- `darkSat/brightSat` 是亮度压缩后的颜色补偿保护，配置不当会让暗部颜色发脏或高亮颜色发灰。
- WDR 下应先确认 WDR merge 和曝光比，再调 DRC；否则可能用 DRC 掩盖了中短帧噪声或接缝源头。

### 典型调试取舍
| 目标 | 常见调整方向 | 图像收益 | 主要副作用 |
| --- | --- | --- | --- |
| 暗部细节更多 | 抬暗部曲线、增强暗区细节 | 背光主体更清楚 | 暗部发灰、噪声变大 |
| 高光层次更好 | 压缩高亮曲线、调亮区抑制 | 天空 / 灯牌不过曝 | 高亮发灰、亮感下降 |
| 画面更通透 | 提高局部对比和细节增益 | 层次和立体感增强 | halo、黑白边、噪声增加 |
| 过渡更自然 | 放大统计窗口、降低滤波权重 | 亮暗交界更平滑 | 局部对比变弱 |

## 调试步骤
1. 参考默认配置，先用 CoolView 的 DRC_CURVE 拉取合适曲线，使整体亮度和对比度达到预期。
2. 固定统计窗和滤波权重后，调整高亮抑制门限和斜率，使亮暗过渡自然。
3. 将 `darkSat` 临时设为 0，观察亮暗分割曲线的作用区域，再恢复到合适值。
4. 根据亮暗区分别调整局部对比和细节增益。
5. 用 `darkSat/brightSat` 修正 DRC 导致的过饱和或颜色分界。
6. DRC curve 和核心参数确定后，`bright` 只作为辅助补偿，通常保持稳定。
7. 各 db 下的细节增益需要和 NR / APC 联调，避免为了通透性放大噪声颗粒。

## WDR 下 DRC 注意点
- 先在宽动态场景确认亮暗分割曲线，再调整其他参数。
- Stat0 主要控制前景 / 正常亮度区域，Stat1 主要控制后景 / 亮部区域。
- 降低 Stat1 值域窗可提升背光人脸亮度、减轻人脸黑边，但会降低人脸对比度。
- 宽动态下暗部 DRC 曲线通常要拉高，亮部要压缩。

## 常见问题入口
- [[wiki/issues/亮度不自然|亮度不自然]]：重点检查 DRC curve、`bright`、滤波权重和亮暗分割。
- [[wiki/issues/暗部发灰|暗部发灰]]：检查暗部曲线是否拉太高、局部对比是否不足、暗部饱和度是否被抑制。
- [[wiki/issues/高光过曝|高光过曝]]：检查亮区抑制门限、DRC 曲线高亮段和 WDR 输出。
- [[wiki/issues/噪声大|噪声大]]：DRC 细节增益、暗部拉升和对比度增强都会放大噪声颗粒。
- [[wiki/issues/锐化过强|锐化过强]]：DRC 细节增益过强会形成白边、黑边或 halo。
