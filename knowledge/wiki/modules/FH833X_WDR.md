# FH833X_WDR

FH833X WDR 模块用于多曝光帧合成，在高动态范围场景中同时保留高亮和暗部细节。

## 页面属性
- 类型：平台模块
- 厂家：Fullhan / 富瀚微
- 平台：[[wiki/platforms/FH833X|FH833X]]
- 模块：WDR / 宽动态
- 场景：两帧合成、三帧合成、Sensor 合成、高光恢复、曝光比、merge 曲线、运动伪影
- 适用范围：指定平台

## 模块作用
- WDR 通过不同曝光时间的长 / 中 / 短帧合成，使画面既能看清亮处细节，也能保留暗部信息。
- FH833X 支持三帧 / 两帧合成和 Sensor 合成相关调试。
- WDR 调试重点是曝光比、长中短帧亮度一致性、merge 曲线和短帧亮度矫正时序。

## 关键参数
### 基础控制
- `wdr_en`：WDR 策略总开关。
- `mode`：宽动态 / 非宽动态模式选择。
- `wdrMode`：manual 或 gainMapping。
- `sensor_output_max`：sensor 最大输出归一化值，通常 0xfff；高亮发粉或发灰时可适当减小。

### 中短帧亮度矫正
- `mf_corr_mode` / `sf_corr_mode`：中帧 / 短帧亮度校正模式。
- `mf_corr_timing` / `sf_corr_timing`：中帧 / 短帧亮度矫正曲线与曝光参数的生效时序匹配。
- `mOffset` / `sOffset`：中帧 / 短帧亮度校正偏移，可降低中短帧暗处噪声，也会影响合成运动区域噪声。

短帧曝光行数随环境变化时，亮度矫正增益也会变化；若曝光和矫正增益未在同一帧生效，高亮区域容易闪烁。

### 曝光比
- `exposure_ratio_ms`：中短帧曝光比。
- `exposure_ratio_lm`：长中帧曝光比。
- 默认常用 0x10，即 16 倍，但必须符合 sensor 支持能力。
- 状态寄存器 `curr_exp_ratio21` / `curr_exp_ratio10` 可查看实际曝光比；实际值受整数行约束，可能与目标值不同。

### Merge 曲线
- `wdr_merge_cfg`：输出配置。
  - 0：三帧合成。
  - 1：只出长帧。
  - 2：只出中帧。
  - 3：只出短帧。
  - 4：长中帧融合。
  - 5：中短帧融合。
- `merge_lm_curve_idx`：长中帧内置 merge 曲线，值越大中帧越早合入。
- `merge_ms_curve_idx`：中短帧内置 merge 曲线，值越大短帧越早合入。
- 自定义曲线通过 `WDR_MERGE_LM_CFG0~7`、`WDR_MERGE_MS_CFG0~7` 配置。

### 高光恢复
- `lf_hlr_en/th/slope`：长帧高光恢复。
- `mf_hlr_en/th/slope`：中帧高光恢复。
- `sf_hlr_en/th/slope`：短帧高光恢复。
- 高亮发粉或发灰可用 HLR 或降低 `sensor_output_max`，手册更推荐降低 `sensor_output_max`。

## 调试视角
WDR 调试的核心不是单独追求“亮处不过曝、暗处看得见”，而是在长 / 中 / 短帧亮度一致性、噪声、运动合成和高光颜色之间取舍。

- 曝光比越大，高光保留能力越强，但中短帧信噪比更差，merge 区域更容易出现噪声、接缝和运动伪影；曝光比过小则高光容易过曝。
- `merge_lm_curve_idx` / `merge_ms_curve_idx` 值越大，中 / 短帧越早合入，高光层次更容易保住，但中短帧噪声和运动错位也会更早进入最终图。
- `mOffset/sOffset` 可以压中短帧暗处噪声，但会改变帧间亮度匹配；压得过多时 merge 区域可能发灰、断层或运动区域噪声形态异常。
- `mf_corr_timing/sf_corr_timing` 与曝光生效帧不匹配时，问题通常表现为高亮区域随 AE 变化闪烁，而不是普通全局曝光闪烁。
- 降低 `sensor_output_max` 能改善高亮发粉 / 发灰，但会压缩高亮端可用范围；HLR 能补局部高光颜色，但过强可能让高光过渡不自然。
- WDR 之后的 DRC、NR、AWB 会放大或掩盖 WDR 问题，所以要先用 `wdr_merge_cfg` 单出各帧确认源头。

### 典型调试取舍
| 目标 | 常见调整方向 | 图像收益 | 主要副作用 |
| --- | --- | --- | --- |
| 高光层次更好 | 增大曝光比，短帧更早合入 | 车灯、天空、灯牌不易爆 | 短帧噪声、接缝、运动伪影增加 |
| 暗部更干净 | 减少中短帧过早合入，配合 NR | 暗部噪声下降 | 高光过渡空间变小 |
| 减少 HDR 接缝 | 调整 merge 曲线和帧间亮度矫正 | 亮暗交界更自然 | 高光或暗部细节可能牺牲 |
| 减少高光闪烁 | 校准 corr timing 和 AE 延迟 | AE 变化时高光稳定 | 需要逐帧验证，调试成本高 |

## 调试步骤
1. 将 `mode` 设为宽动态模式。
2. 选择内置 merge 曲线，并按 sensor 规格配置长中短帧曝光比，常用固定 16 倍。
3. 用 `wdr_merge_cfg` 单出长 / 中 / 短帧，确认各帧亮度和噪声状态。
4. 若长短帧经亮度矫正后亮度不一致，检查 sensor 库曝光实现和曝光比配置。
5. 输出合成帧，把 AE 改为手动曝光，观察高亮处是否随曝光变化闪烁。
6. 如果高亮闪烁，调整 `sf_corr_timing` / `mf_corr_timing`，匹配曝光和亮度矫正时序。
7. 根据高亮保留、暗部细节和运动伪影需求选择或自定义 merge 曲线。
8. 室外高亮环境可限制 AE 最小曝光时间为 16 行或更多，避免亮度过暗。
9. 高亮发粉 / 发灰时，优先考虑降低 `sensor_output_max`，必要时再调 HLR。
