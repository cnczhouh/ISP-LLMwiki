# FH833X_SHARP_NOISE

FH833X SHARP_NOISE 章节是锐化与降噪联调方法，用于在静止清晰度、运动噪声、拖影和低照噪声之间做系统权衡。

## 页面属性
- 类型：平台模块 / 联调流程
- 厂家：Fullhan / 富瀚微
- 平台：[[wiki/platforms/FH833X|FH833X]]
- 模块：NR3D / NR2D / YNR / APC / CNR / DPC / GB
- 场景：锐化降噪联调、运动噪声、拖影、监控强 3DNR、车载弱 3DNR
- 适用范围：指定平台

## 核心思路
FH833X 手册把锐化与降噪联调分为两类方案：

- 强 NR3D 降噪方案：常见于监控或车内监控，摄像机相对静止，目标运动。
- 弱 3DNR 降噪方案：常见于车载，摄像机相对周边场景运动，更重视拖影和运动清晰度。

两类方案都需要先完成颜色、亮度、NR3D 等基础模块，再联调 NR2D、YNR、APC、CNR。

## 强 NR3D 方案
适用场景：摄像机静止，目标运动。目标是在静止区域干净清晰的同时，控制运动区域噪声和拖影。

### 调试顺序
1. 调 APC 基础锐化。
2. 调 NR2D 运动噪声 / 空域降噪。
3. 调 YNR 二次亮度降噪。
4. 调 APC coeff 联动，降低大运动物体锐化。
5. 调 CNR 色度噪声。
6. 优化运动噪声抹除范围和 NR3D 拖影。

### 关键要点
- APC 基础锐化前可 bypass DPC、GB、NR2D、YNR 等影响清晰度模块。
- NR2D 负责静止区域和小运动区域抹干净，并保留大运动区域部分噪声形态。
- YNR 对 NR2D 后噪声做二次降噪，同时通过 coeff 联动区分静止、小运动、大运动。
- APC 联动建议主要减弱大运动区域锐化，不建议把小运动到大运动全部减弱，否则容易全局模糊。
- CNR 通过 `str`、颜色阈值和 NR3D coeff 联动控制运动区域色噪。

## 弱 3DNR 方案
适用场景：车载应用，摄像机相对环境运动。目标是优先保证运动清晰度和低拖影，再控制噪声。

### 调试顺序
1. 先关闭或弱化 NR3D，模拟车辆行进时全屏运动。
2. 调 APC 基础锐化，低照下偏向描大边而不是强化细颗粒纹理。
3. NR2D 只去除一部分噪声，不宜过强。
4. YNR 再去除一部分亮度噪声，每级降噪都避免过强。
5. APC 可先在关闭 3DNR 时做全局优化，最后再用 APC 联动优化车辆静止场景。
6. CNR 按色噪和运动区域情况补充。

### 关键要点
- 弱 3DNR 场景中，车辆行进时几乎全局大运动，应优先保证运动清晰度。
- YNR 的 coeff 联动阈值可按照度调节；低照时阈值和噪声回加策略需同步变化。
- APC coeff 权重最大值通常保持 255 保证静止区清晰度，最小值随低照逐步减弱以避免运动区噪声太大。

## 拖影调优
- 暗区拖影较重：优先减小 `offset`，减弱暗区时域降噪强度。
- 亮区拖影较重：优先减小 `slope`，减弱亮区时域降噪强度。
- 低照亮度区分度不明显且拖影严重时，需要同时减小 `slope` 和 `offset`。
- 还可调整 `mdgain`、`refCoeffRatio`、`nr3dCofSel`、`coeffMode`、`refCoeffTh`、`coeffMapIdx` 等 NR3D 参数控制拖影和背景噪声释放。
